01Overview

A capstone in the Beckhoff ecosystem.

This project is a graduate coursework assignment for EE 5340 — Programmable Logic Controllers, Lab Exercise 6. The objective was to gain working familiarity with Beckhoff's Windows-based PLC platform and the TwinCAT 3 development environment, and to use that environment to design, download, and commission a sequential control program for a realistic industrial process: a cereal box filling station.

The lab builds up gradually — starting from basic combinational logic (a series AND circuit and a parallel OR circuit), adding a TON timer exercise, and culminating in a full machine-control program that sequences a conveyor motor, a fill solenoid, and a pair of sensors across five discrete production steps with stop / start pause-resume behavior and a manual reset. The deliverable is a running ladder program, commissioned against physical lamps and switches wired into the CX5140's digital I/O terminals.

02Hardware & I/O Map

CX5140 on the rail, EtherCAT to the bench.

The target controller is a Beckhoff CX5140 — a DIN-rail-mounted embedded industrial PC running the TwinCAT real-time runtime alongside a standard Windows image. Digital input and output terminals are scanned into the project automatically and addressed via IEC physical-address format. Five physical pushbuttons and switches and five indicator lamps are wired onto a bench board to simulate the plant signals of the cereal filler.

03Process Description

Five steps, two dwells, one latched run bit.

The cereal filler operates as a five-step sequence. On initial start-up, conveyor motor M1 runs until an empty box reaches the fill position, which is detected by limit switch LS. The machine then pauses for 2 seconds to let the box settle before opening the FILL solenoid. Fill continues until the weight sensor WT_SENSE registers a full box, at which point the solenoid closes and the sequence waits another 3 seconds before restarting the conveyor to eject the full box and bring the next empty one into position.

Motor M2 runs continuously whenever the machine is in the RUN state. If Stop_PB is pressed at any time, the machine pauses at the current step — motors and fill solenoid are dropped but the internal timers continue to run, so that pressing Start_PB resumes the suspended step cleanly. A separate Reset_PB clears all steps so that the next Start_PB press begins the cycle from scratch; per the specification the reset input is ignored while the machine is actively running.

04Program Networks

Set/reset state machine in ladder.

Rather than nested latches or timer chains, the program is organized as a classic five-step sequential function model implemented in ladder. Five Boolean flags (STEP1 through STEP5) mark which phase the machine is in at any given scan, a master RUN bit gates all outputs, and transitions between steps are implemented as set/reset pairs driven by sensor inputs and timer .Q outputs.

01
Supervisor
Run latch — seal-in on Start_PB, breakdown on Stop_PB
A standard seal-in circuit. When RUN drops, all outputs go low but the step bits and timers are preserved — pressing Start_PB resumes exactly where the sequence left off. Pause-resume falls out of the design naturally.
02
Init Detection
No-steps-active helper
A series of negated STEP1..STEP5 contacts produces a helper bit used when initializing the first step on a clean start.
03
Transition
STEP1 → STEP2 on limit switch
When LS asserts (box in position), STEP1 is reset and STEP2 is set, beginning the 2-second settle dwell.
04
Transition
STEP2 → STEP3 after 2 s
TON_1 with a T#2s preset runs while STEP2 is active; when TON_1.Q asserts, STEP2 is reset and STEP3 is set, gating the fill solenoid.
05
Transition
STEP3 → STEP4 on weight sensor
When WT_SENSE asserts (box full), STEP3 is reset and STEP4 is set, dropping the fill solenoid.
06
Transition
STEP4 → STEP5 after 3 s
TON_2 with a T#3s preset runs while STEP4 is active; when TON_2.Q asserts, STEP4 is reset and STEP5 is set, restarting the conveyor for eject.
07
Transition
STEP5 → STEP1 when box clears
When LS deasserts (box has moved off the sensor), STEP5 is reset and STEP1 is set, beginning the next cycle.
08
Reset Path
Reset_PB clears all steps — only with RUN off
Reset_PB AND NOT RUN resets every step bit so the next Start_PB press begins from scratch. Reset is intentionally ignored while the machine is running.
09
Output
M1 conveyor — STEP1 OR STEP5
M1 = (STEP1 OR STEP5) AND RUN — the conveyor runs during the "search for empty box" phase and the "eject full box" phase, gated by the master run bit.
10
Output
M2 continuous feed
M2 = RUN — the product feed runs continuously while the machine is enabled, regardless of step.
11
Output
FILL solenoid — STEP3 only
FILL = STEP3 AND RUN — the solenoid only energizes during the fill phase. This structure is directly analogous to a PackML-style execute sub-state machine implemented at the scale of a single unit procedure.
05TwinCAT Commissioning

A typical Beckhoff commissioning day.

The lab also served as a walk-through of Beckhoff's commissioning tooling, which differs noticeably from other PLC vendors. The path from empty solution to running ladder on real hardware follows a clean three-stage workflow.

01
Route / Connect

Secure ADS over Ethernet

Development laptop connected to the CX5140 over Ethernet with the controller IP entered manually into TwinCAT's "Choose Target" dialog. A Secure ADS route added with the device fingerprint copied via right-click → Copy Fingerprint and pasted into the "Compare with" dialog to prevent man-in-the-middle attacks.

02
Scan / Map

EtherCAT terminal discovery

TwinCAT placed in Config mode. right-click Devices → Scan populates every EtherCAT terminal on the DIN rail automatically — markedly cleaner than Rockwell's Ethernet discovery because EtherCAT carries device identification natively. Variables declared as AT %I* / AT %Q* are then bound to physical channels via Change Link….

03
Activate / Login

Online verification & force

Activate Configuration with Autostart checked, accept the trial license, then PLC → Login. Energized contacts and coils highlight blue in the ladder view. Inputs that aren't on the bench (like Reset_PB) are exercised via the Force function on the channel's Online tab.

07Outcome

Five steps, two timers, online on real hardware.

The final program runs on the CX5140 and correctly sequences all five production steps — conveyor motor until limit switch, 2-second settle dwell, fill until weight sensor, 3-second eject dwell, and return — with correct pause-and-resume behavior on the stop pushbutton and a full state reset via the dedicated reset input. Beyond the mechanics of the specific filler, the lab was a concrete walkthrough of what a typical Beckhoff / TwinCAT commissioning day looks like end-to-end: bring up the project, pick the target, route securely over ADS, scan the EtherCAT terminals, write and link the logic, and go on-line to verify against real hardware. That workflow is directly transferable to any TwinCAT-based machine build.
